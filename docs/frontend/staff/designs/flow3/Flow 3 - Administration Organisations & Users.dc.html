DECISION — open question 1 (tree depth handling): recommending flat indentation (28px per level) over collapsible levels. Herit's org hierarchy is a government structure — realistically a handful of levels, not hundreds of nodes — so collapse/expand state management is unneeded complexity for the common case. Cost of the alternative: collapsible levels scale better to a pathological deep tree, at the price of persisting expand state and an extra click to reach any deep node on every visit.

DECISION — open question 2 (deleting yourself / the last admin): the backend does not prevent either case — verified, no guard exists server-side. Recommending a client-side guard: block the delete action here with an explanation, rather than letting the request go through and lock everyone out of user administration. This is a UI safeguard only; it doesn't close the backend gap, which should still be fixed server-side.
